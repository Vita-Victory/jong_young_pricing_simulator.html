## 📈 Methodology

### **Data Collection**
- Web scraping: Python (Selenium, BeautifulSoup)
- Source: Google Maps verified reviews
- Period: 12 months
- Sample: 36 reviews (anonymized)

### **Sentiment Analysis**
- NLP classification: Positive (4-5★), Neutral (3★), Negative (1-2★)
- Tool: Claude AI API
- Accuracy: ~95% (validated on sample)

### **Theme Extraction**
- Automated topic identification + manual coding
- 8 themes extracted and categorized
- Frequency-ranked by customer mention count

### **Business Modeling**
- Revenue: Product pricing × monthly volume × seasonality
- Costs: COGS + labour + rent + marketing + growth investments
- Profit: Gross revenue - total costs
- Scenarios: Conservative (80%), Moderate (100%), Aggressive (120%)

### **Validation**
- Cost estimates vs. industry benchmarks ✓
- Pricing vs. local competitors ✓
- Seasonality vs. retail patterns ✓
- Model tested with edge cases ✓

## ⚠️ Limitations & Next Steps

### **Current Limitations**
- Sample size: 36 reviews (50+ would increase confidence)
- Cost estimates: Based on benchmarks (should validate with actual financials)
- Price-demand: Assumed linear (may be non-linear)
- ±10-15% accuracy range (good for strategy, not precision)

### **To Enhance Analysis**
- Integrate actual order data & transaction history
- Add competitive pricing analysis
- Track recommendation implementation & actual ROI
- Monthly refresh with new review data
- A/B test pricing scenarios
